```python
def align_semantic_similarity(text_a: str, text_b: str) -> float:
    """
    返回 0-1 的相似度分数
    
    流程:
    1. 优先查询本地同义词表 (O(1) 查找)
    2. 如果不在词表中 → 使用向量相似度 (快速)
    3. 如果向量相似度在 [0.65-0.75] 的灰度区间,
       只有此时才调用 LLM 进行精细判定
    """
    
    # 第一层: 同义词表查询 (成本 $0)
    if (text_a, text_b) in SYNONYM_DICT:
        return SYNONYM_DICT[(text_a, text_b)]
    
    # 第二层: 向量相似度 (成本 $0, 使用本地模型)
    vec_sim = get_vector_similarity(text_a, text_b)
    
    if vec_sim > 0.75:  # 高相似度，直接判定为同义
        return 0.95
    elif vec_sim < 0.65:  # 低相似度，直接判定为不同义
        return 0.1
    else:  # 灰度区间 [0.65-0.75]，调用 LLM
        # 成本: ~$0.001/call
        # 预计出现次数: <1% 的比较
        llm_sim = llm_semantic_check(text_a, text_b)
        return llm_sim
```

### 性能指标

```

  → 同义词表命中 → 返回 0.95 (< 1ms)

  → 向量相似度 0.82 → 返回 0.82 (< 20ms)
```

## 决策 3: 元数据来源

**选择**: **③ 混合模式** (本地优先，API 补充)

### 分层策略

```
数据优先级:

优先级 1 (本地,零成本): P1 已有数据
├─ 年份: 从 entity_timeline 获取
├─ 文献 ID: 从检索结果获取
├─ 来源文献 (文件名): 从原始 chunks 元数据

优先级 2 (向量库,~$0.01/call): 本地模型提取
├─ 文献标题: 从 chunks 的段落标记
├─ 作者初步识别: 规则提取 (Name, et al.)
├─ 材料/工艺标记: NER 提取

优先级 3 (外部 API,~$0.1/call): CrossRef/Scrapedia
├─ 正式作者列表
├─ 通讯作者
├─ 影响因子 (年份特定)
├─ 被引计数
├─ DOI/期刊信息

优先级 4 (可选,高成本): 全文爬虫
├─ 机构信息
├─ 关键词
├─ 摘要验证
```

### 成本控制

```
场景分析:

常规冲突 (冲突数 < 10):
  → 仅使用优先级 1+2 (成本 $0)
  
中等冲突 (冲突数 10-50):
  → 对高关键冲突 (severity >= 3) 调用 API (成本 $5-10)
  
严重冲突 (冲突数 > 50):
  → 批量调用 API (成本 $15-20)

预计成本:
  100 条 eval_queries → ~5-10 个高级冲突
  → 调用 API 次数: 5-10 次
  → 成本: ~$1-2 USD
  
加上 LLM 语义判定:
  ~1-2% 的比较 × 50K claims = 500-1000 次 LLM 调用
  → 成本: ~$5-10 USD
  
总成本: ~$6-12 USD (在 <$20 预算内)
```
