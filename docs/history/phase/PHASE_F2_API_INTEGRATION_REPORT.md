## API Documentation

### Endpoint: GET /recovery/events

**Purpose:** Retrieve event timeline for recovery inspection

**Query Parameters:**
- `session_id` (optional): Filter by writing session ID
- `job_id` (optional): Filter by job ID
- `time_filter` (optional): Reserved for future time-range filtering

**Responses:**
- **200 OK:** EventTimelinePayload with all matching events
- **500 Error:** If retrieval fails

**Example Request:**
```bash
GET /recovery/events?session_id=sess_001&job_id=job_001
```

**Example Response:**
```json
{
  "events": [
    {
      "event_id": "evt_001",
      "event_type": "JOB_STARTED",
      "timestamp": "2026-04-10T12:00:00Z",
      "source_job_id": "job_001",
      "source_session_id": "sess_001",
      "event_data": {"status": "started"}
    }
  ],
  "event_count": 1,
  "start_time": "2026-04-10T12:00:00Z",
  "end_time": "2026-04-10T12:05:00Z",
  "session_filter": "sess_001",
  "job_filter": "job_001"
}
```

---

### Endpoint: GET /recovery/memory

**Purpose:** Get current memory state snapshot

**Responses:**
- **200 OK:** MemorySnapshotPayload with all current facts
- **500 Error:** If retrieval fails

**Example Request:**
```bash
GET /recovery/memory
```

**Example Response:**
```json
{
  "facts": [
    {
      "fact_id": "fact_001",
      "namespace": "execution",
      "subject": "job_001",
      "predicate": "status",
      "object": "completed",
      "object_type": "string",
      "valid_from": "2026-04-10T12:00:00Z",
      "valid_to": null,
      "source_event_id": "evt_001"
    }
  ],
  "fact_count": 1,
  "namespaces": ["execution"],
  "last_updated": "2026-04-10T12:00:00Z"
}
```

---

### Endpoint: POST /recovery/facts/invalidate

**Purpose:** Invalidate a fact by closing its validity window

**Request Body (InvalidFactRequest):**
```json
{
  "fact_id": "fact_001",
  "namespace": "execution",
  "reason": "Incorrect state",
  "invalidated_by": "user_001"
}
```

**Responses:**
- **200 OK:** FactInvalidationPayload confirming invalidation
- **400 Bad Request:** Missing required fields or validation error
- **500 Error:** If operation fails

**Example Response:**
```json
{
  "fact_id": "fact_001",
  "namespace": "execution",
  "reason": "Incorrect state",
  "previous_value": "running",
  "invalidated_at": "2026-04-10T12:05:00Z",
  "invalidated_by": "user_001",
  "success": true
}
```
