# History Archive

This directory stores historical and non-canonical repository documents that should no longer live in the repository root.

## Directory map

- `phase/`: phase delivery reports, phase prompts, and phase summaries
- `harness/`: harness-specific plans, reports, and completion notes
- `reports/`: project reports, delivery reports, implementation status reports, and final handoff reports
- `plans/`: one-off plans, roadmaps, quick references, and prompt bundles
- `sessions/`: session summaries and conversation completion notes
- `diagnostics/`: raw text outputs such as error logs, compile logs, and verification output

## Archive rules

- Historical files stay here unless they are promoted into an active canonical document.
- Root-level active documentation should remain limited to stable guides such as `README.md`, `GETTING_STARTED.md`, `DEVELOPER_GUIDE.md`, `ARCHITECTURE.md`, and `NAMING_AND_ARCHIVE_POLICY.md`.
- New archived files should follow the naming guidance in [NAMING_AND_ARCHIVE_POLICY.md](/C:/Users/xiao/Desktop/tools/Modular-Pipeline-Script/NAMING_AND_ARCHIVE_POLICY.md).
