# 金标集需求清单（用户版，简洁）

**日期**: 2026-04-25  
**目标**: 100–200 条查询的文献检索金标集，支持论文相关性分级  
**格式**: JSONL (JSON Lines)  

---

## 核心 7 字段（每条查询必须有）

| 字段 | 类型 | 示例 | 说明 |
|------|------|------|------|
| `schema_version` | 字符串 | `"1"` | 固定值 `"1"` |
| `query_id` | 字符串 | `"q_g0042"` | 唯一 ID，建议 `q_g<4位数字>` |
| `query_text` | 字符串 | "激光功率如何影响焊缝穿透深度?" | 自然语言查询 |
| `qrels` | 数组 | `[{doc_id, relevance, source_hint}, ...]` | 相关文献列表 |
| `annotator_id` | 字符串 | `"xiao"` | 标注者 ID |
| `no_gold` | 布尔 | `false` | 无相关文献时为 `true` |
| `created_at` | 字符串 | `"2026-04-25T12:00:00Z"` | ISO 时间戳 |

---

## 相关性分级（qrels 中的 relevance）

- **2 = 直接相关** — 文献直接回答问题或提供核心证据
- **1 = 边缘相关** — 文献涉及话题但不直接，或仅提及
- **0 = 无关** — 文献不能回答问题

---

## 覆盖维度（想测试的查询类型）

| 类型 | 数量 | 示例 |
|------|------|------|
| **机制题** | 15–20 | "深穿透激光焊的钥匙孔模式如何形成?" |
| **参数-结果** | 25–30 | "激光功率从 2kW 升到 3kW，穿透深度如何变化?" |
| **对比题** | 20–25 | "脉冲激光 vs 连续激光在热影响区大小上有何差异?" |
| **数据提取** | 15–20 | "从图表读出 200mm/min 速度下焊缝抗张强度" |
| **综述题** | 15–20 | "汽车工业激光-混合焊接行业最佳实践是什么?" |
| **小众** | 5–10 | 领域特定、罕见现象 |

---

## 分层标签（source_stratum）

每条查询必须标注 **一个**：

| 层 | 数量 | 描述 |
|----|------|------|
| **S1** | ~20 | 简单关键词匹配（"激光焊接"、"材料性能"） |
| **S2** | ~40 | 中等合成（对比、多因素分析） |
| **S3** | ~30 | 复杂跨论文推理（综合 5+ 篇） |
| **S4** | ~10–20 | 自由探索、长尾问题 |

---

## 快速标注流程

### 第 1 步：准备查询池
- 使用模板或自由书写
- 列出 100–200 条自然语言问题

### 第 2 步：对每条查询
1. **搜索语料库** (BM25、embedding，或手工阅读)
2. **找出相关论文**，标记相关性 (0/1/2)
3. **记录来源** (`bm25`, `dense`, `evidence_set` 等)
4. **填补必需字段** (query_id, annotator_id, pool_size 等)
5. **填可选字段** (source_stratum 最重要)

### 第 3 步：格式化 & 验证
```bash
python gateb_schema_validator.py your_goldset.jsonl
```

### 第 4 步：交付
- 无错误的 JSONL 文件 ✅

## 完成标准

- ✅ 通过 schema validator (零错误)
- ✅ S1 + S2 ≥ 60%, S3 ≥ 20%, S4 ≥ 5%
- ✅ 每条查询考虑 ≥ 10 个候选 (pool_size)
- ✅ 相关性分布均衡（避免全 rel=2 或全 rel=0）
- ✅ ≤ 5% 的查询无相关文献 (no_gold=true)
